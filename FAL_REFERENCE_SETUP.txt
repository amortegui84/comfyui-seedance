FAL reference-to-video setup

Folder reviewed:
D:\ComfyUI_windows_portable_nvidia\ComfyUI_windows_portable\ComfyUI\custom_nodes\fal-flux-nodes

How fal-flux-nodes reads the API key

1. First choice: config.ini in the fal-flux-nodes folder
2. Second choice: environment variable FAL_KEY

Relevant files

- fal-flux-nodes\modules\base_fal_node.py
- fal-flux-nodes\config.ini

What to do

Option A: keep the key in fal-flux-nodes\config.ini

[fal]
api_key=YOUR_FAL_KEY_HERE

Option B: use an environment variable instead

PowerShell:
$env:FAL_KEY="YOUR_FAL_KEY_HERE"

Important notes

- Do not commit your real FAL key into git
- If a real key was ever exposed in a tracked file or shared output, rotate it in the fal.ai dashboard
- fal-flux-nodes sets FAL_KEY automatically after loading the key

About Seedance and references

The fal-flux-nodes pack you have includes:
- fal-ai/bytedance/seedance/v1/lite/reference-to-video

That node accepts:
- prompt
- 1 to 4 reference images
- aspect_ratio
- resolution
- duration
- camera_fixed
- seed
- enable_safety_checker

Procedure in ComfyUI for fal-flux-nodes

1. Make sure your FAL key is set in config.ini or FAL_KEY
2. Restart ComfyUI
3. Add the Seedance Lite reference-to-video node from fal-flux-nodes
4. Connect at least 1 reference image
5. Enter the prompt
6. Run the workflow

Important limitation

- This fal-flux-nodes pack is using Seedance v1 Lite reference-to-video
- It is not the same as the Seedance 2.0 real-human asset flow
- It does not use your human asset_id/group_id from Seedance AM

About Seedance AM in this repo

If you use Seedance AM with provider = fal.ai:
- only the fal API key is needed in the Seedance AM - API Key node
- base_url is ignored for fal.ai
- reference images/video/audio can be used
- human_asset_id is not supported on fal.ai in this repo

Recommended separation

- Use fal-flux-nodes for fal native reference-to-video with normal reference images
- Use Seedance AM + AnyFast when you need real-human asset_id/group_id flows
